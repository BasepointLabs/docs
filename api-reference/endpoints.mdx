This page summarizes the main customer-facing API routes. Admin-only operator routes are intentionally excluded unless they explain a customer-visible product flow.

<Note>
  Normal customer routes use the signed-in session cookie and active workspace. They are not bearer-token API routes today.
</Note>

## Health and waitlist

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/health` | Check service health |
| `POST` | `/waitlist` | Join the marketing waitlist |

| `GET` | `/agent-templates` | List AI employee templates |
